### 추천: **A — 태스크 완료/상태 변경 (BL-005)**

### 추천 A — 정상 흐름 & 오류 유형
**정상 흐름(1줄)**: 사용자가 자기 태스크의 "완료" 버튼을 누르면 `status`가 `done`으로 바뀌고 목록에 즉시 반영된다.

| # | 오류 유형 | 원인 | 처리 |
|---|---|---|---|
| 1 | 미인증 | 로그아웃 상태에서 호출 | `/login` 리다이렉트, 액션은 "로그인 필요" 거부 |
| 2 | 권한 없음 | 타인(다른 `created_by`) 태스크 변경 시도 | UPDATE RLS(`auth.uid()=created_by`)로 0건 → "변경 권한 없음" |
| 3 | 대상 없음 | 존재하지 않는/이미 사라진 task id | 영향 행 0 → "태스크를 찾을 수 없음" |
| 4 | 잘못된 상태값 | 허용 외 status('todo'/'doing'/'done' 아님) | 서버 검증에서 거부(현재 DB CHECK 없음 → 액션에서 화이트리스트 검증) |
| 5 | 저장 실패 | 네트워크·DB 오류 | `console.error`(서버 로그) + "잠시 후 다시 시도" 메시지 |
