### 추천 A — 정상 흐름 & 오류 유형
**정상 흐름(1줄)**: 사용자가 자기 태스크의 "완료" 버튼을 누르면 `status`가 `done`으로 바뀌고 목록에 즉시 반영된다.

| # | 오류 유형 | 원인 | 처리 |
|---|---|---|---|
| 1 | 미인증 | 로그아웃 상태에서 호출 | `/login` 리다이렉트, 액션은 "로그인 필요" 거부 |
| 2 | 권한 없음 | 타인(다른 `created_by`) 태스크 변경 시도 | UPDATE RLS(`auth.uid()=created_by`)로 0건 → "변경 권한 없음" |
| 3 | 대상 없음 | 존재하지 않는/이미 사라진 task id | 영향 행 0 → "태스크를 찾을 수 없음" |
| 4 | 잘못된 상태값 | 허용 외 status('todo'/'doing'/'done' 아님) | 서버 검증에서 거부(현재 DB CHECK 없음 → 액션에서 화이트리스트 검증) |
| 5 | 저장 실패 | 네트워크·DB 오류 | `console.error`(서버 로그) + "잠시 후 다시 시도" 메시지 |

## 2. 오류 메시지 가이드

대상 기능: **태스크 상태 변경(완료)** — 흐름: 완료 클릭 → 서버 액션(getUser → status 검증 → UPDATE tasks WHERE id & created_by=auth.uid() → revalidate).
규칙: 사용자 메시지엔 stack trace·SQL·키 금지(무슨 일 + 어떻게 + 안내만). 개발자 로그엔 `XXX-숫자` 오류 코드 필수.

| 오류 유형 | 상황 | 사용자 메시지(한국어) | 개발자 로그 | 재시도 정책 |
|---|---|---|---|---|
| 입력: 잘못된 상태값 | 허용 외 status('todo'/'doing'/'done' 아님)가 들어옴 | 상태 값이 올바르지 않습니다. 화면의 버튼으로 다시 시도해 주세요. | `TASK-422 invalid status value=<redacted>` | 재시도 불가(입력 교정) |
| 입력: 대상 미지정 | formData에 task id가 없음(잘못된 요청) | 변경할 태스크가 지정되지 않았습니다. 목록에서 다시 선택해 주세요. | `TASK-400 missing taskId` | 재시도 불가(재선택) |
| 대상 없음 | 존재하지 않거나 사라진 task id → 영향 행 0 | 해당 태스크를 찾을 수 없습니다. 목록을 새로고침해 주세요. | `TASK-404 task not found (0 rows)` | 재시도 불가(새로고침) |
| 권한 없음 | 타인(다른 created_by) 태스크 변경 시도 → RLS로 0건 | 이 태스크를 변경할 권한이 없습니다. 본인이 만든 태스크만 변경할 수 있어요. | `RLS-403 update denied (not owner)` | 재시도 불가 |
| 미인증/세션 만료 | getUser 결과 없음(로그아웃·토큰 만료) | 로그인이 필요합니다. 다시 로그인한 뒤 시도해 주세요. | `AUTH-401 no authenticated user` | 재로그인 후 1회 재시도 |
| 동시성 충돌 | 다른 탭/세션이 먼저 변경해 상태가 이미 바뀜 | 이미 다른 곳에서 변경된 태스크입니다. 새로고침 후 확인해 주세요. | `TASK-409 stale state on update` | 새로고침 후 재시도 |
| 내부 저장 실패 | DB·네트워크 일시 오류 | 일시적인 오류로 저장하지 못했습니다. 잠시 후 다시 시도해 주세요. | `DB-500 update failed code=<pg_code>` | 사용자 수동 재시도(1~2회) |

> 사용자 메시지엔 코드·SQL·키 미노출. 개발자 로그의 `<redacted>`·`<pg_code>`도 값 원문이 아니라 분류용 표식만 남긴다.
