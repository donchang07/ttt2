# Day10 핸드오프 (A-09-O4)

작성일: 2026-06-28 (Day09 산출물 사후 보완) · 근거: [mcp_candidate_decision.md](./mcp_candidate_decision.md), [mcp_test_log.md](./mcp_test_log.md)
Day10 Slice 2(태스크 배정/우선순위)로 넘기는 6개 요소 + 각 폴백 전략.

| # | 요소 | 내용 | 폴백 전략 |
|---|---|---|---|
| 1 | **Slice2용 도구** | `list_recent_tasks`(읽기 전용)로 최근 태스크를 읽어 우선순위 추천 초안 입력으로 사용 | 도구 미등록 시 `dist/index.js`를 stdio로 직접 구동, 그래도 안 되면 더미 목록 수기 입력 |
| 2 | **인증 오류** | MCP는 로컬 stdio라 자체 인증 없음. 앱 측 Slice2는 `getUser()` 미인증 시 AUTH-401 | 미인증=배정/추천 차단, 로그인 유도. 도구 응답을 신뢰 말고 앱에서 세션 재확인 |
| 3 | **경로 오류** | 서버 빌드 산출물 경로 `ttt2/mcp/dist/index.js`(레포 밖) | 경로 누락 시 `npm run build` 재실행으로 재생성, 절대경로로 호출 |
| 4 | **권한 오류** | 도구가 쓰기 권한 없음(읽기 전용). 앱 Slice2의 배정은 leader/team_id RLS로 제한 | 타 팀·비leader 시도는 RLS 0행/거부 → 거짓 성공 방지(TASK-404 처리) |
| 5 | **외부 도구 실패** | MCP 서버 미기동·tools/call 실패 | 추천 기능은 도구 의존이므로, 실패 시 "추천 불가" 명시하고 수동 우선순위로 폴백(흰 화면 금지) |
| 6 | **다음 기능 후보** | Slice 2 = `assignTask` 서버 액션 + AssignForm(담당자 배정) | AI 추천(Claude 키)이 막히면 배정 기능만 먼저 출시하고 추천은 Day12/13으로 분리 |

## 요약
Day10은 읽기 전용 MCP 출력을 **입력으로만** 쓰고, 모든 권한·격리는 앱 서버 액션과 RLS에서 재확인한다. 외부 도구 실패는 기능 차단이 아니라 수동 폴백으로 흡수.
