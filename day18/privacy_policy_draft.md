# 개인정보 처리방침 초안 — TaskFlow (Day18 산출물 A-18-O3)

> **수업용 초안 — 게시 전 법률 검토 필요.** 실제 서비스 게시 전 반드시 전문가(변호사·개인정보보호 담당) 검토를 받아야 합니다.

작성일: 2026-06-28 · 서비스: TaskFlow(팀 태스크 관리 SaaS, AI 우선순위 추천).

## 처리 항목 표
| 요소 | 내용 |
|---|---|
| **수집 항목** | 이메일(로그인 식별), 표시 이름(display_name), 서비스 이용 로그(`activity_logs`: 이벤트 타입·시각·ID), 팀/태스크/노트 데이터, RAG용 업로드 문서(텍스트·PDF) |
| **수집 목적** | 계정 인증·세션 유지, 팀 태스크 관리 기능 제공, AI 우선순위·문서 질의 응답, 서비스 운영·오류 진단 |
| **보관 기간** | 계정 유지 동안 보관. 탈퇴/삭제 요청 시 지체 없이 파기. 운영 로그(`activity_logs`)는 ID·이벤트만 보관(개인 식별정보 미포함) |
| **삭제 방법** | 관리자 콘솔의 사용자 삭제(`admin_delete_user`)로 계정·그가 만든 태스크·문서까지 일괄 삭제. 데이터는 Supabase에서 제거 |
| **제3자 처리(위탁)** | Supabase(데이터 저장·인증, 호스팅), Vercel(애플리케이션 호스팅), Anthropic Claude(AI 답변 생성), OpenAI(문서 임베딩). 각 사 정책에 따라 처리, 원문·PII 최소 전송 |
| **문의 연락처** | (게시 전 기입: 개인정보 보호책임자 이메일/연락처) |

## 처리 원칙
- 운영 로그·알림에는 이메일·이름·전화·문서 원문·키를 보내지 않음(ID·이벤트·에러코드만, 보안 게이트 마스킹).
- 데이터는 팀/사용자 단위 RLS로 격리. 제3자(AI) 전송은 기능 수행에 필요한 최소 범위.
