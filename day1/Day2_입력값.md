# Day 02 시장분석 입력값 (A-01-O4)

작성일: 2026-06-27 · Day 01 산출 → Day 02 검색 대상

## 1. 경쟁사 후보 5개

**직접 경쟁 (팀 태스크/이슈 관리)**
1. **Asana** — 팀 작업 관리, 타임라인·보드. 우선순위 기능은 수동
2. **Linear** — 스타트업·개발팀 이슈 트래커. 빠른 UX, 키보드 중심
3. **Trello** — 칸반 보드, 진입장벽 낮음. 우선순위 근거 제시는 없음

**간접 경쟁 (대체 수단)**
4. **Notion** — 문서+DB로 태스크를 직접 구성. 자유롭지만 세팅 부담
5. **Slack 채널 + 메모** — 별도 도구 없이 대화로 우선순위 합의

## 2. 고객 가설 3개 (누가 · 어떤 상황에서 · 왜)

1. **누가**: 5~15인 스타트업 PM이 **어떤 상황에서**: 매일 아침 스탠드업 직전 **왜**: 오늘 우선순위를 빠르게 정해 팀에 공유해야 해서
2. **누가**: 팀리드가 **어떤 상황에서**: 태스크가 30개 넘게 쌓였을 때 **왜**: 무엇부터 할지 근거 있게 가려내야 해서
3. **누가**: 팀원이 **어떤 상황에서**: 여러 요청이 동시에 들어올 때 **왜**: 내 작업 우선순위의 근거를 확인하고 싶어서

## 3. 시장 확인 질문 3개

1. 소규모 팀은 우선순위 결정에 실제로 얼마의 시간/마찰을 겪는가? (정량)
2. 기존 도구(Asana·Linear·Notion)의 우선순위 기능에 대한 불만 지점은 무엇인가?
3. "AI가 제시한 우선순위 근거"를 팀이 신뢰하고 받아들이는가?

## 4. 차별화 가설

- 경쟁 도구는 우선순위를 **사람이 수동 정렬**한다.
- TaskFlow는 **AI가 우선순위 + 근거(왜 이 순서인지)를 30초 안에 제시**하고, 팀이 그 근거를 보고 합의한다.
- 핵심 차별점: "정렬"이 아니라 "**근거 있는 추천**" + team 단위 격리로 안전한 협업.
