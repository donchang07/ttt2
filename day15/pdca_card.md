# PDCA 카드 (Day15 산출물 A-15-O2)

작성일: 2026-06-28 · 대상 문제: **로그인 후 발견성**(T1·T4) — 로그인했는데 앱/관리자 화면으로 가는 길이 불명확.

## Plan (무엇을·왜)
- **가설**: 로그인 사용자가 홈(`/`, 마케팅)에 도착하면 앱 메뉴를 못 봐서 "화면이 없다"고 느낀다.
- **계획**: 홈을 서버 컴포넌트로 바꿔, 로그인 상태면 `/tasks`(상단 메뉴 포함)로 리다이렉트. 로그아웃 사용자는 기존 랜딩 유지(SEO·전환 보존).

## Do (코드 변경)
- `src/app/page.tsx`: `getUser()` 추가 → `user`면 `redirect("/tasks")`. 1파일, 최소 변경.
- 커밋: `fix: redirect logged-in users from home to app (user test reflection)`.

## Check (before → after 증거)
| 상태 | 동작 |
|---|---|
| **Before** | 로그인 사용자가 `/` 방문 → 마케팅 랜딩 표시("시작하기/로그인"만). 앱·관리자 메뉴 안 보임 → "관리자 화면이 없다"(T1). |
| **After** | 로그인 사용자가 `/` 방문 → 즉시 `/tasks`로 이동, 상단에 태스크·노트·(관리자) 메뉴 노출. 길을 헤매지 않음. |
- 검증(prod 실측, 2026-06-28): 로그아웃 `/` → **200**(랜딩 유지, SEO·전환 보존), 미인증 `/tasks` → **307 /login**. 로그인 사용자 `/` → `/tasks` 리다이렉트는 `src/app/page.tsx`의 `getUser()` 가드(`if (user) redirect("/tasks")`)로 보장(배포본 반영, 커밋 `e58e45e`).
