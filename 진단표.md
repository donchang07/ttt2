# TaskFlow 프로젝트 진단표 (A-01-O1)

작성일: 2026-06-27 · Day 01

## 1. 제품 진단 — 누구의 어떤 문제

- **대상 사용자**: 5~15인 규모 스타트업/소규모 팀의 PM·팀리드
- **핵심 문제**: 매일 "오늘 무엇을 먼저 할지" 우선순위 결정에 평균 15분 이상 소모. 근거가 사람 머릿속에만 있어 팀 공유가 안 됨
- **해결 가설**: 팀 태스크를 한곳에 모으고, AI가 우선순위 근거를 30초 안에 제시
- **핵심 가치**: 결정 시간 단축 + 우선순위 근거의 가시화·공유

## 4. 4주 목표 (Must / Should / Won't)

**Must (반드시)**
1. 도메인 스키마(team·member·task·note) + 마이그레이션
2. 모든 테이블 RLS 정책 (team 단위 격리)
3. 이메일 인증 로그인
4. 태스크 CRUD + 목록/상세 UI
5. AI 우선순위 추천 (근거 텍스트 포함)

**Should (가능하면)**
1. 관리자 콘솔(멤버/역할 관리)
2. 알림(Slack Webhook)
3. 사용자 행동 분석

**Won't (이번엔 안 함)**
1. 모바일 네이티브 앱
2. 결제/구독
3. 다국어
4. 외부 캘린더 연동

## 5. 측정 가능한 성공 기준

- 우선순위 결정 시간: 15분 → 3분 이하 (체감 측정)
- 신규 태스크 등록 후 AI 추천까지 응답 5초 이내
- 빌드·typecheck·lint 항상 통과 상태 유지
- 모든 테이블 RLS 적용률 100%

## 6. 가장 큰 리스크 1개

- **보안(데이터 격리)**: 멀티테넌트 SaaS에서 RLS 미설정 시 타 팀 데이터 노출. 테이블 생성과 동시에 RLS를 의무화한다. (Day 04·Day 11에서 집중)
